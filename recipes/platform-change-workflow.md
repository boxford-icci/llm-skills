# Platform change workflow

Use this for schema changes, API contract shifts, auth-sensitive work, or distributed-system rollout risk.

## Suggested sequence

1. `The Architect`
2. `The Security Engineer`
3. `The Migration Engineer`
4. `The Contract Tester`
5. `The Reliability Engineer`
6. `The Enforcer`

## Copy-paste prompt

```text
Use The Orchestrator to route this platform change. I want the smallest useful lineup that covers architecture, security, migration safety, contract compatibility, reliability, and final ship risk. Bias toward Architect, Security Engineer, Migration Engineer, Contract Tester, Reliability Engineer, and Enforcer.
```
